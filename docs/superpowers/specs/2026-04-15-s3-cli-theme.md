# S3 — Visual CLI theme (pastel green)

**Date**: 2026-04-15
**Sub-project**: 3 of 5 (see `2026-04-14-cli-daemon-overview.md`)
**Branch**: `006-cli-theme`
**Status**: Drafted, awaiting user review
**Depends on**: none (parallelisable with S1)
**Target version**: `0.3.0` (same bump as S2/S4/S5)

## Goal

Give the CLI a coherent visual identity aligned with the loomflo logo (pastel green / Mint palette), centralise all terminal output through a single theme module, and respect the standard terminal signals (`NO_COLOR`, non-TTY, `--json`).

## Non-goals

- Frontend / dashboard theming → **S5** (Tailwind tokens extend the same palette there, but that's handled in S5's scope).
- ASCII art / banners / intro splash-screens → out of scope; clean typography wins.
- Full-screen TUI with curses-style rendering → out of scope (basic `ora` + `cli-table3` only).
- Replacing `commander` with an alternate CLI framework → no.

## Palette (Mint — validated 2026-04-15)

| Token | Hex | RGB | Role |
|---|---|---|---|
| `accent` | `#8BD1B5` | `139,209,181` | Success, active nodes, primary highlights |
| `muted` | `#A7D7C5` | `167,215,197` | Labels, secondary text, project names |
| `dim` | `#6B7A78` | `107,122,120` | Metadata (timestamps, pids, costs) |
| `warn` | `#E6C97A` | `230,201,122` | Retries, non-fatal warnings |
| `err` | `#E8908C` | `232,144,140` | Failures, fatal errors |

Status glyphs: `✓` (success), `✗` (fail), `→` (progress), `⚠` (warn), `●` (dot for statuses in tables).

## Module API

```ts
// packages/cli/src/theme.ts
import chalk from "chalk";

export const palette = { accent: [139,209,181], muted: [167,215,197], dim: [107,122,120], warn: [230,201,122], err: [232,144,140] } as const;

export const theme = {
  accent: (s: string) => chalk.rgb(...palette.accent)(s),
  muted:  (s: string) => chalk.rgb(...palette.muted)(s),
  dim:    (s: string) => chalk.rgb(...palette.dim)(s),
  warn:   (s: string) => chalk.rgb(...palette.warn)(s),
  err:    (s: string) => chalk.rgb(...palette.err)(s),

  // glyphs
  check: "✓", cross: "✗", arrow: "→", warnGlyph: "⚠", dot: "●",

  // layout helpers
  heading(text: string): string,           // underlined accent title
  kv(key: string, value: string): string,  // "  key  value" aligned
  line(glyph: string, tone: Tone, text: string, meta?: string): string,  // "✓ text  (meta)"
  spinner(text: string): Ora,              // pre-configured ora instance
  table<T>(headers: string[], rows: T[], columns: Column<T>[]): string,  // cli-table3 wrapper
};
```

`chalk@5` is chosen over `picocolors` because the pastel palette requires 24-bit truecolor (`chalk.rgb()`). Chalk gracefully degrades to the closest 256-color or 16-color approximation on terminals without truecolor support (via `chalk.level` auto-detection) and natively respects `NO_COLOR`, `FORCE_COLOR`, and non-TTY. Tradeoff: ~10 KB added to the CLI bundle versus picocolors' ~0.5 KB — acceptable for a dev tool.

All output in commands must flow through `theme.*`. No raw `console.log` with hardcoded colors anywhere in `packages/cli/src/commands/`.

### No-color fallback

`theme` relies on `chalk`'s built-in detection (`chalk.level`):
- `process.env.NO_COLOR` truthy → `chalk.level = 0`, all color fns return the input unchanged.
- `!process.stdout.isTTY` → same (unless `FORCE_COLOR` explicitly set).
- `process.env.FORCE_COLOR === "0"` → disabled.
- Truecolor unavailable → downgrades to 256-color or 16-color ANSI.

Layout helpers (`heading`, `table`) check `chalk.level === 0` to render ASCII-only borders when colors are fully off.

### `--json` flag

A shared `--json` option is added to every user-facing command (`init`, `start`, `status`, `logs`, `project`, `daemon`, `config`, and the new S4 commands). When present:
- Theme helpers are bypassed entirely.
- Command prints a single newline-terminated JSON object/array to stdout.
- No spinners, no progress, no banners.
- Errors go to stderr as `{ "error": "...", "code": "..." }`.

Enforcement: a shared `withJsonSupport(cmd)` wrapper in `packages/cli/src/output.ts` — commands opt in by calling it; grep rule added to CI to flag `console.log` in commands directory.

## Migration scope

Refactor the following files to route all output through `theme.*`:

- `packages/cli/src/commands/init.ts`, `start.ts`, `stop.ts`, `status.ts`, `resume.ts`, `chat.ts`, `logs.ts`, `daemon.ts`, `project.ts`, `config.ts`, `dashboard.ts`.
- `packages/cli/src/client.ts` error paths (currently raw `console.error`).
- `packages/cli/src/index.ts` help-text overrides (use `theme.accent` for command names in the banner).

Before/after sample:

```ts
// Before
console.log(`✓ daemon running (pid ${pid}, up ${uptime}s)`);

// After
console.log(theme.line(theme.check, "accent", "daemon running", `pid ${pid}, up ${uptime}s`));
```

## Dependencies added

```json
{ "chalk": "^5.x", "ora": "^8.x", "cli-table3": "^0.6.x" }
```

All three are established and actively maintained. `chalk@5` and `ora@8` are ESM-only; the CLI is bundled with `tsup` which handles the ESM→CJS transform for the final distributable, so downstream consumers see no difference.

## Testing strategy

- **Unit** (`packages/cli/test/theme.test.ts`):
  - Snapshot `theme.heading`, `theme.kv`, `theme.line`, `theme.table` with `chalk.level` pinned to `3` (truecolor) and to `0` (no color).
  - Verify `NO_COLOR=1` / `chalk.level=0` returns ANSI-free strings.
  - Verify `--json` path bypasses theme entirely (spy on `process.stdout.write`).
- **Regression**: existing command integration tests must still pass (output formatting changes; tests that grep for specific strings may need adjustment — update them to tolerate ANSI codes via `strip-ansi` helper).

## Success criteria

- Every CLI command's output uses the 5-token palette consistently.
- `NO_COLOR=1 loomflo start` emits ANSI-free output.
- `loomflo <any-cmd> --json` emits parseable JSON, no colors, no spinners.
- No `console.log` with raw strings remains in `packages/cli/src/commands/`.
- Visual QA: comparison screenshot of a `start` run before/after matches the palette mockup shown during brainstorming.

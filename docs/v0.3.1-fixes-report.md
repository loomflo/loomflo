## Fix Details

### 1. Exponential backoff + non-retryable error bail-out (R1 + R2)

**Files**:
- `packages/cli/src/onboarding/index.ts`
- `packages/cli/src/onboarding/validators.ts`
- `packages/cli/tests/unit/onboarding/retry.test.ts`

**Changes**:
- Added `RETRY_BASE_MS = 500` constant. Before each retry attempt > 1, the
  loop now awaits `500 * 2^(attempt-2)` ms, producing delays of 500ms and
  1000ms for a 3-attempt sequence.
- Extended `ValidatorResult` with an optional `retryable?: boolean` field.
- All three validators (`anthropic-oauth`, `anthropic`, `openai-compat`) now
  set `retryable: false` for 401/403 auth errors and local token failures,
  and `retryable: true` for network exceptions and 5xx server errors.
- The retry loop in `runExistingValidator` breaks immediately when
  `res.retryable === false`, avoiding wasted spinner cycles on deterministic
  failures.
- Two new tests added: "bails immediately on non-retryable errors" and
  "applies exponential backoff between retries" (intercepting setTimeout to
  verify delay values).

### 2. Timing-safe token comparison (R3)

**File**: `packages/core/src/api/server.ts`

**Changes**:
- Added `safeTokenEquals(a, b)` helper using `crypto.timingSafeEqual`.
  Handles differing-length strings by performing a dummy constant-time
  comparison before returning `false`, so length information is not leaked.
- Replaced `presentedToken !== token` (WebSocket auth, line 460) with
  `!presentedToken || !safeTokenEquals(presentedToken, token)`.
- Replaced `header.slice(BEARER_PREFIX.length) !== token` (HTTP Bearer auth)
  with `!safeTokenEquals(header.slice(BEARER_PREFIX.length), token)`.

### 3. Breaking change documentation (R4)

**File**: `CHANGELOG.md`

**Changes**:
- Added a "Breaking changes" section to the 0.3.0 entry documenting the
  migration from `?token=` query string to `Sec-WebSocket-Protocol`.
- Includes before/after code snippets for migrating WebSocket clients.
- Notes that old clients will be rejected with close code 4001.

### 4. config.json 0600 permissions (R5)

**Files**:
- `packages/cli/src/commands/init.ts`
- `packages/cli/tests/unit/onboarding/init.test.ts`

**Changes**:
- `writeFile` for config.json now passes `mode: 0o600`.
- Unconditional `chmod(configFile, 0o600)` follows the write, matching the
  pattern established by P0-3 for project.json.
- New test: "writes config.json with 0600 mode (R5)" asserts file permissions
  after init.

### 5. Cold-start E2E test (R6)

**File**: `packages/core/tests/unit/cold-start.test.ts`

**Changes**:
- New test case: "full cold-start chain: daemon starts, stub profile is usable
  by the wizard". Verifies:
  1. `daemon.start()` creates the stub `default` profile.
  2. `ProviderProfiles.get("default")` resolves with `{ type: "anthropic-oauth" }`.
  3. The profile shape has exactly one key (`type`), matching the discriminated
     union.
  4. Unauthenticated requests to API routes return 401.
  5. Authenticated `GET /daemon/status` returns 200 with `projectCount: 0`.
  6. `POST /projects` with the stub profile's name registers a project (201).
